Tags Plugin

Requirements :

* jquery(Required)
* Server (for ajax calls)
* bootstrap(Optional)

Usage :

Simple to implement.

An Input field with an "id".
<input type="text" name="tags" id="tags">

Scripts to trigger.
<script>
$("#tags").tags({
url:"browsers.json",
/* some optional options */
method:"post",
data:{"_token":"xyz"},
call:"prefetch",
return_val:"id",
image:true,
strict:false,
existing_val:[{"id":"2","image":"images/firefox.gif","tag":"firefox"},{"id":"1","image":"images/chrome.gif","tag":"chrome"}], repeat:true,
nomatch:"press comma to proceed !",
});
<script>

Options :


url :

url argument takes the url which return the json data.And Url is a required element,if url is not valid or failed to pass as an sarugument a Javascript alert will be thrown.


method :

method option takes the http method type of the above given url.

Default : "get" method

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"get" (or) method:"post" ,
call:"prefetch",
});
</script>


call :

call options states the way of communication with the given url whether it should be frequently or Once.

The two types of Calls are
prefetch
remote

prefetch :

prefetch loads the required data once in your page.

remote :

remote loads the url whenever the user enter data into the input field.And the user input value will be send to the server as "q" variable.
(For example an user enter "hello" into input field in remote call, he can get the user entered variable as "q=hello")

Default : "prefetch"

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"get",
call:"prefetch", or call:"remote"
});
</script>


data :

data holds some json data to be send along with the given url request.

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"post" ,
call:"prefetch",
data:{token:"xyz"}
});
</script>


return_val :

return val carries the data to be returned as value.

Default : "id"

Usage :

someurl.json :
[{"id":1,"image":"test.jpg","tag":"text"},{"id":2,"image":"test2.jpg","tag":"text2"}]

<script>
$("#tags").tags({
url: "someurl.json",
method:"get",
call:"prefetch",
return_val:"tag",
});
</script>

Output :
value="text","text2"


image :

image is the boolean option.If "true" it will show images in both suggestions and tags.

Default : "false"

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"post" ,
call:"prefetch",
data:{token:"xyz"},
image:true, (or) image:false
});
</script>


strict :

image is also boolean option.If "true" ,it allows user to add their own tags bye pressing comma(,) rather from url contents.

Default : "false"

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"post" ,
call:"prefetch",
data:{token:"xyz"},
image:true,
strict:true,
return_val:"id", });
</script>


repeat :

image is also a boolean option.If "true" ,it allows user to add same tags again and again.

Default : "false"

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"post" ,
call:"prefetch",
data:{token:"xyz"},
image:true,
strict:true,
repeat:true, (or) repeat:false,
return_val:"id", });
</script>


existing :

existing_val holds the json data,which should be loaded as exisiting value in input.

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"post" ,
call:"prefetch",
data:{token:"xyz"},
image:true,
strict:true,
existing_val:[{"id":1,"image":"test.jpg","tag":"text"}],
return_val:"id",
});
</script>

Output(while loading of page itself) :
value="1"


nomatch :

nomatch carries the error data to be shown if there is nomatch between user given data and json data.

Deafult : "No match Found"

Usage :

<script>
$("#tags").tags({
url: "someurl",
method:"post" ,
call:"prefetch",
data:{token:"xyz"},
image:true,
strict:true,
nomatch:"press comma to add new",
});
</script>


Json Format :

Please provide the response as the following json format for proper execution

Image : true
[{"id":1,"image":"test.jpg","tag":"text"},{"id":2,"image":"test2.jpg","tag":"text2"}]

Image : false
[{"id":1,"tag":"text"},{"id":2,"tag":"text2"}]
